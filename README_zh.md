# FicForge · 粮坊

[English](README.md) | **中文**

> 冷圈磕 CP，全世界只有一个半人嗑这对。
> 想自己产粮，但 AI 写着写着就忘了角色是谁、伏笔埋了什么、上一章发生了什么。
> 
> 所以我做了粮坊。

FicForge 是给**写冷 CP、用 AI 产粮**的人做的工具。

不是通用 AI 写作助手，不是小说生成器。是给那些找不到粮、只能自己动手的人，提供一个**让 AI 写到第 100 章还记得第 5 章设定**的系统。

---

## 它解决什么问题

用 AI 写长篇同人，你大概遇到过这些：

- **崩人设**：写了 3000 字的角色分析，AI 下一章就忘了他是什么性格
- **丢伏笔**：精心铺了三章的伏笔，AI 当没发生过
- **断联系**：角色关系发展到某个阶段了，AI 突然倒退回初见的状态
- **忘设定**：你设定了一个完整的世界观，AI 完全不管，自己瞎编

根本原因是 AI 的上下文窗口有限——它不可能记住你写过的每一个字。市面上的 AI 写作工具试图用"自动摘要"解决这个问题，但摘要会丢信息，丢的还是你觉得最重要的那些细节。

FicForge 的做法不同：**你来决定什么重要，AI 负责在写作时精准注入。**

---

## 它怎么工作

- **剧情笔记**（Plot Points）：你标记"这是一个伏笔"，它就会一直留在 AI 的记忆里，直到你说"已填坑"
- **角色设定**：核心性格特征有不可挤占的"低保预算"——即使上下文快满了，主角的人设也不会被裁掉
- **本章重点**：每一章你可以指定"这章必须推进这条线"，AI 不会自由发挥跑偏
- **AI 设定助手**：用大白话告诉 AI"帮我建一个角色"，它会给你建议，你逐条确认后才生效——AI 只建议，不擅自动手
- **语义搜索**：AI 写作时自动检索你的设定和历史章节中最相关的片段，不需要你手动复制粘贴
- **多稿对比**：同一章生成多个版本，翻页对比，选最满意的那个定稿

## 隐私

- **所有数据保存在你自己的电脑上。** 粮坊不会把你的设定、章节、任何东西上传到我们的服务器——我们没有服务器
- 当你使用 AI 写作功能时，内容会发送到你自己配置的 API 提供商（如 DeepSeek）——这是 AI 生成必需的，完全由你控制
- 内置的语义搜索在本地运行，不经过任何云端

## 安装

### Windows

1. 从 [Releases](../../releases) 下载最新版本
2. 运行安装包
3. 打开粮坊 → 配置 API 密钥（推荐 [DeepSeek](https://platform.deepseek.com)，便宜好用）→ 开始写

### 从源码构建

```bash
# 后端
cd src-python
pip install -r requirements.txt
PYTHONPATH=. python main.py

# 前端
cd src-ui
npm install
npm run dev
```

需要 Python 3.12+ 和 Node.js 18+。

## 兼容性

- **模型**：支持所有 OpenAI 兼容 API——DeepSeek、GPT、Ollama、本地模型
- **语言**：中英双语界面，随时切换
- **导入**：支持 txt / md / docx / html，自动切分章节
- **内置搜索**：自带中文向量模型（bge-small-zh），开箱即用。英文写作用户建议在全局设置中配置 API Embedding 以获得更好的检索效果

## 负责任使用

粮坊是给自己产粮用的，不是拿来抄别人的。

- 请只导入你自己的作品
- 发布 AI 辅助创作的内容时，建议标注 AI 的参与
- 导出功能默认附加 AI 辅助声明
- 请遵守你所在创作社区的规则
