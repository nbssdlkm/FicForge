# FicForge 用户引导设计评估

---

## 现有设计（已经不错的部分）

### ✅ 10分钟成功路径（PRD §1.5）
有明确的时间约束：安装<2分钟 → 首次续写<5分钟 → 理解核心功能<10分钟。
这是很好的设计约束，但它更像一个验收标准，不是具体的引导流程设计。

### ✅ 新建AU三选一引导（补充PRD §1.16）
"用AI配置 / 从原作导入 / 直接写" 三个选择清晰，照顾了不同用户偏好。

### ✅ 导入后能力分层说明（PRD §1.6）
"导入 ≠ 系统理解全文" 的预期管理做得好，三层表格很清晰。

### ✅ 导入后冷启动引导
导入完成后引导提取最近5-20章facts，有明确的选项和时间预估。

### ✅ 设定模式首次切换提示（补充PRD §1.3）
一次性 tooltip 解释设定模式是什么。

## 缺失或薄弱的部分

### 🔴 1. 首次启动流程几乎是空白

PRD 只写了："弹出欢迎引导：填写 API Key → 测试连接 → 进入首页"，一句话带过。

**用户第一次打开应用，看到什么？** 没有设计。需要补充：

```
建议流程：

1. 欢迎页（不是设置页！）
   "欢迎使用 FicForge · 粮坊"
   "AI 帮你写同人，记住每一个设定，不再崩人设。"
   [开始设置 →]

2. API 配置（引导式，不是表单页）
   "首先，连接你的 AI 模型"
   "推荐 DeepSeek，便宜好用，注册就送额度"
   [我有 API Key] → 填Key + 测试
   [还没有？→ 跳转注册教程链接]
   [我用 Ollama 本地模型] → 切换到 Ollama 配置

3. 创建第一个 Fandom
   "你想写哪个作品的同人？"
   [输入框：如 底特律：变人、原神、咒术回战...]
   或 [跳过，稍后创建]

4. 进入首页
   如果创了Fandom → 直接引导创建第一个AU
   如果跳过 → 首页显示空状态引导
```

### 🔴 2. 空状态设计完全没有

用户在各种"还没有内容"的页面会看到什么？PRD 没有定义。

```
需要设计的空状态：

首页（无Fandom时）：
  "还没有作品，创建你的第一个 Fandom 开始写文吧"
  [+ 创建 Fandom]  [📥 导入旧文]

AU设定集（无角色时）：
  "还没有角色设定"
  "添加角色后，AI 续写时会自动参考人设，避免崩人设。"
  [+ 添加角色]  [📥 从原作导入]  [🤖 让AI帮你整理]

剧情笔记（无记录时）：
  "还没有剧情笔记"
  "记下重要剧情和伏笔，AI 会帮你记住并在续写时参考。"
  "写了几章后，点击「提取」让 AI 自动从正文中总结。"
  [+ 手动记一笔]  [✨ 从已有章节提取]

写作底线（无条目时）：
  "还没有写作底线"
  "写作底线是 AI 绝对不能违反的规则，比如："
  "「两人目前还没有在一起」「角色 A 不知道角色 B 的秘密」"
  [+ 添加一条底线]

章节列表（无章节时）：
  "还没有章节，去写作页开始第一章吧"
  [✍️ 开始写作]
```

### 🔴 3. Fandom / AU 两层结构没有解释

这是最容易让新用户困惑的概念。一个写手打开应用，
不理解为什么要先建"原作"再建"AU"。

```
建议：在创建 Fandom 时加一句引导

"FicForge 用「原作」和「AU」两层来管理你的同人设定。

原作 = 你写的是哪个作品（比如底特律、原神）
       放角色的原始人格分析、作品世界观等通用资料

AU   = 你这篇文的具体设定（比如星际皇室AU、现代咖啡馆AU）
       放这篇文里角色的具体形象、独特世界观、剧情笔记

一个原作下可以有多个 AU，共享原作资料但互不干扰。
不想分层？直接建AU写就行，原作资料可以以后再补。"
```

### 🟡 4. 核心功能没有渐进式引导

facts（剧情笔记）、chapter_focus（本章重点）、pinned_context（写作底线）
是 FicForge 最强大也最容易被忽略的功能。但现在的设计是：
全部一次性展示在界面上，用户看到但不知道怎么用。

```
建议：里程碑式触发引导

写完第1章确认后：
  "第一章写好了！继续写下去吧。"
  （不引导任何高级功能，让用户先感受核心体验）

写完第3章确认后：
  "你已经写了3章了！现在可以试试「剧情笔记」——
   记下重要剧情和伏笔，AI 续写时会自动参考。
   [✨ 从最近3章自动提取]  [稍后再说]"

写完第5章确认后（如果还没添加底线）：
  "写了5章了，有没有什么 AI 绝对不能搞错的规则？
   比如角色关系、秘密、性格底线？
   添加到「写作底线」，AI 每次续写都会严格遵守。
   [➕ 添加底线]  [不需要]"

第一次有 unresolved fact 时：
  "💡 你有一个待填的坑：「{fact内容}」
   写下一章时，可以把它设为「本章要推的线」，
   AI 会在这章重点推进这个剧情。
   [设为本章重点]  [先自由发挥]"
```

### 🟡 5. "直接开始写作"路径太薄

用户选了"直接开始写作"后，只有侧边栏一句话提示
"还没有配置角色和设定，AI续写质量可能不佳"。

这句话的问题：
- 负面语气（"质量可能不佳"），用户觉得被指责
- 没告诉用户什么时候该加设定、加了有什么好处

```
建议改为更正面的引导：

首次续写前（无设定时）：
  输入框上方温和提示：
  "💡 还没有角色设定？没关系，先写起来！
   随时切到「改设定」模式添加角色，AI 就能帮你记住人设了。"

续写结果出来后（如果出现OOC或明显不一致）：
  不做自动检测（太复杂），但在 Context 面板空 P5 层旁标注：
  "核心设定：空（添加角色设定后，AI 每次续写都会参考）"
```

### 🟡 6. 错误信息不够友好

PRD 定义了6种错误类型（超时/429/402/context超限/安全拦截/Key无效），
但只定义了 error_code 和 actions，没有设计用户看到的具体文案。

```
建议补充错误文案表：

| 错误 | 用户看到的文案 |
|------|-------------|
| API Key 无效 | "密钥验证失败，请检查你的 Key 是否正确" |
| 429 限流 | "请求太频繁了，等 {N} 秒后再试" |
| 402 额度不足 | "API 额度用完了，请去服务商充值" |
| 超时 | "AI 想太久了没回应，再试一次？" |
| context 超限 | "这章的设定太多了，AI 读不完。试试减少一些角色设定或缩短底线。" |
| 安全拦截 | "AI 服务商拒绝了这次请求（可能触发了内容审查），换个说法试试？" |
| 连接失败 | "连不上 AI 服务，检查一下网络和接口地址？" |
```

### 🟡 7. 设定模式的"确认卡片"缺引导

补充PRD设计了确认卡片的UI，但没有解释用户第一次看到卡片时怎么理解。

```
建议：首次出现确认卡片时加一句说明

"以下是 AI 建议的设定变更，每条可以单独确认或编辑。
 确认后才会真正保存，你也可以全部取消。"
```

## 总结：需要补充的引导设计

| 优先级 | 内容 | 工作量 |
|--------|------|--------|
| 🔴 高 | 首次启动欢迎引导流程 | 前端 1-2天 |
| 🔴 高 | 所有页面的空状态设计 | 前端 1天 |
| 🔴 高 | Fandom/AU 两层结构解释 | 文案 + 前端半天 |
| 🟡 中 | 里程碑触发式功能引导 | 前端 1-2天 |
| 🟡 中 | 错误信息友好文案表 | 文案半天 |
| 🟡 低 | 确认卡片首次引导 | 前端半天 |
| 🟡 低 | "直接写作"路径的正面引导 | 文案 + 前端半天 |

**核心原则**：不要一次性教完所有功能。
让用户先成功写出第一章，再逐步引导高级功能。
