# FicForge UI 术语改名方案

> 原则：用同人圈太太日常用语，不用开发术语，不用AI行业术语，不用英文（除非圈内公认如 AU/OC）。
> 目标用户画像：LOFTER/AO3 写文太太，可能完全不懂技术。
> 技术选项原则：凡是普通写手不可能自己搞懂的技术概念（向量、embedding、token、RAG、context window），
> 要么不暴露，要么用比喻 + 一句话解释 + 合理的默认值，让用户"不改也能用，想改看得懂"。

## 五、系统/技术选项（用户不该看到或需要大幅简化）

| 当前 | 问题 | 建议 |
|------|------|------|
| embedding_lock | 纯技术 | 用户不需要看到这个词。显示为 **"搜索引擎模型"** + 引导文字："AI 用这个模型来查找和你写的内容相关的设定。换了模型需要重新处理一遍所有设定，大约需要几分钟。" |
| index_status: ready/stale/rebuilding | 纯技术 | 简化为图标 + 人话：✅"设定已就绪" / ⚠️"设定需要重新处理" / 🔄"正在处理设定…" |
| context_window | 半技术 | **一次能读多少字** + 引导文字："AI 每次续写时能参考的最大文字量。数字越大，AI 能记住的前文和设定越多。大部分模型是 128k（约10万字），一般不用改。" |
| token_warning_threshold | 纯技术 | 隐藏，不暴露给普通用户 |
| schema_version | 纯技术 | 不暴露 |
| sync_unsafe | 纯技术 | 不暴露，只在出问题时弹窗用人话解释 |
| ops.jsonl | 纯技术 | 永远不暴露 |
| content_hash | 纯技术 | 不暴露 |
| api_base | 半技术 | **接口地址** + 引导文字："你的 AI 服务商提供的地址，比如 DeepSeek 就填 https://api.deepseek.com" |
| api_key | 半技术 | **密钥** + 引导文字："在你的 AI 服务商网站上获取，类似一串密码，用来验证你的身份" |
| model_params_override | 纯技术 | 显示为"本篇专属参数调整" |
| Embedding（出现在设置页） | 技术术语 | 标题改为 **"搜索引擎"** + 副标题："用来从你的设定里找到和当前剧情最相关的内容" |
| mode: api / local / ollama | 半技术 | **在线API / 本地模型 / Ollama** + 各自一行说明："在线API：最简单，填Key就能用" / "本地模型：不联网，需要自己下载模型文件" / "Ollama：如果你已经在用 Ollama，可以直接接入" |
| ChromaDB（如果出现在任何提示中） | 纯技术 | 永远不暴露给用户，内部错误日志里可以有 |
| RAG（如果出现在任何提示中） | 纯技术 | 永远不暴露，用"智能检索"或"自动查找相关设定"代替 |

## 改动范围说明

**纯前端改动**（改 i18n / 组件文案）：
- 所有导航名、按钮文案、表单 label、提示文字、引导说明
- 枚举值的显示名（后端枚举值不变，前端做映射）

**不改后端**：
- API 字段名保持英文（perspective、importance、narrative_weight 等）
- 文件名保持英文（characters/、worldbuilding/）
- 内部概念保持不变（cast_registry、pinned_context 等）

**原则：后端说英文，前端说人话。**
