# FicForge 多设备同步指南

## 概述

FicForge 支持桌面端和移动端之间同步数据，让你在电脑上建库、手机上续粮。

## 方案一：文件夹同步（推荐，最简单）

### 原理

将 AU 数据目录放在云盘同步文件夹内（OneDrive / 坚果云 / iCloud），各设备通过云盘客户端同步文件。在全局设置 → 数据同步中点击「立即同步」手动触发合并。

### 步骤

1. 在电脑上安装坚果云 / OneDrive 客户端
2. 将 FicForge 数据目录移动到云盘同步文件夹内
   - Windows: 将 `%APPDATA%/FicForge/data/` 移到 `OneDrive/FicForge/`
   - 或在 FicForge 全局设置中修改数据目录路径
3. 手机上安装坚果云 / OneDrive App
4. FicForge 移动端设置中指定同一个同步目录
5. 完成！每次需要同步时，在全局设置 → 数据同步中点击「立即同步」

### 注意事项

- 确保云盘客户端设置为"始终保留在此设备"（关闭按需下载）
- 避免两端同时编辑同一个文件（虽然有冲突处理，但手动解决麻烦）
- 向量索引不同步——移动端首次打开 AU 时会后台重建（约 1-2 分钟）

## 方案二：WebDAV 同步

### 原理

FicForge 内置 WebDAV 客户端，直接连接支持 WebDAV 的云存储服务。适合手机上没有云盘客户端的用户。

### 支持的 WebDAV 服务

- **坚果云**（推荐，国内最稳定）：`https://dav.jianguoyun.com/dav/`
- **Nextcloud**（自建）：`https://你的域名/remote.php/dav/files/用户名/`
- **群晖 NAS**：`https://NAS地址:5006/`
- **TeraCloud**：`https://nanao.teracloud.jp/dav/`

### 步骤

1. 打开 FicForge → 全局设置 → 数据同步
2. 同步方式选择"WebDAV"
3. 填写服务器地址、用户名、密码
4. 点"测试连接"确认成功
5. 点"立即同步"进行首次同步
6. 每次需要同步时点击"立即同步"

### 坚果云配置步骤

1. 下载并安装坚果云客户端（桌面端或手机端均可）
2. 登录坚果云账号
3. 进入 设置 → 第三方应用管理
4. 点击"添加应用密码"，名称填"FicForge"
5. 记下生成的应用密码
6. 在 FicForge 中：
   - 服务器地址: `https://dav.jianguoyun.com/dav/`
   - 用户名: 你的坚果云账号（邮箱）
   - 密码: 刚才生成的应用密码（不是登录密码）
   - 远程目录: `/FicForge/`

## 同步了什么？

| 数据 | 同步？ | 说明 |
|------|--------|------|
| 章节正文 | Yes | |
| 角色/世界观设定 | Yes | |
| 剧情笔记(Facts) | Yes | 通过操作日志同步，保证一致性 |
| 操作历史 | Yes | 用于合并两端的修改 |
| 草稿 | No | 草稿是临时的，不同步 |
| 全局设置 | No | API Key 等配置每设备独立 |
| 搜索索引 | No | 每端本地自动重建 |

## 遇到冲突怎么办？

如果两端同时修改了同一个文件，FicForge 会弹出冲突解决界面，让你选择保留哪个版本。操作历史（剧情笔记等）会自动合并，不需要手动处理。

## 常见问题

**Q: 手机上打开 AU 提示"正在后台建立搜索索引"？**

A: 正常现象。搜索索引不跨设备同步，每端需要本地重建（约 1-2 分钟）。重建期间可以正常写作，只是智能检索暂时不可用。

**Q: 坚果云免费版够用吗？**

A: 够用。FicForge 一个 AU 的数据量约 50-100MB，坚果云免费版每月 1G 上传 / 3G 下载。而且坚果云支持增量同步，实际传输量很小。

**Q: 可以用百度网盘 / 阿里云盘吗？**

A: 不行。这些网盘不支持 WebDAV 协议，也没有真正的文件夹同步功能。推荐使用坚果云（国内）或 OneDrive（国际）。
